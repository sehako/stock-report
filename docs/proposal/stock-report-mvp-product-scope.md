# Stock Reports Lab MVP 제품 범위

상태: 초안
작성 기준: `docs/proposal` 기획 자료

## 1. 문서 목적

이 문서는 Stock Reports Lab MVP에서 구현할 기능, 구현하지 않는 기능, 추후 확장 기능을 제품 범위 관점으로 정리한다.

구현 세부사항, DB 스키마, API URL, 배치 내부 상태 전이, 인프라 구성은 이 문서의 범위가 아니다.

## 2. 구현할 기능

1. 매 거래일 장 마감 이후 일별 장 마감 리포트를 생성한다.
2. KOSPI와 KOSDAQ 시장 개요를 제공한다.
3. KOSPI와 KOSDAQ 시장 지수 일봉 차트를 제공한다.
4. 거래량 상위 분석 종목 기준 시장 통계를 제공한다.
5. 거래일별 시장 전체 AI 요약 1건을 제공한다.
6. KRX 종목 중 종가 1,000원 이상인 종목을 대상으로 분석한다.
7. 분석 대상은 거래량 상위 200개 종목으로 제한한다.
8. 최근 세 거래 일봉 안에 발생한 Stoch MACD 골든크로스 종목 목록을 제공한다.
9. 골든크로스 종목에 대해 신호 발생일, 현재가, K값, D값, 업종을 표시한다.
10. 해당 거래일의 전체 분석 종목 200개 목록을 제공한다.
11. 분석 종목 목록에서 종목별 가격, 거래량, 분석 상태, 최근 신호를 표시한다.
12. 분석 종목 목록에서 검색과 정렬을 제공한다.
13. 종목 상세 화면을 제공한다.
14. 종목 상세 화면에서 1W, 1M, 3M, 1Y 일봉 캔들 차트를 제공한다.
15. 종목 상세 화면에서 거래량, MACD, Stoch MACD 시계열을 제공한다.
16. 종목 상세 화면에서 골든크로스 발생 마커를 표시한다.
17. 과거 리포트에서 종목 상세로 진입한 경우 해당 리포트 거래일까지의 데이터만 표시한다.
18. FinanceDataReader의 KRX 설명 목록 기준 업종 정보를 사용한다.
19. 분석 종목 200개 기준 업종 현황 화면을 제공한다.
20. 업종 현황 화면에서 업종 트리맵을 제공한다.
21. 업종별 종목 수, 최근 골든크로스 종목 수, 골든크로스 발생 비율을 제공한다.
22. 서비스 운영 시작일 이후의 과거 장 마감 리포트를 거래일별로 조회할 수 있게 한다.
23. 사용자에게는 거래일별 최종 활성 리비전만 노출한다.
24. 한 종목의 수집 또는 분석 실패가 전체 리포트 공개를 막지 않게 한다.
25. 종목별 분석 상태를 사용자에게 구분해 표시한다.
26. AI 요약 생성 실패 또는 지연 중에도 검증된 가격, 차트, 지표, 신호는 제공한다.
27. 관심종목은 브라우저 로컬 저장소에만 저장한다.
28. 모든 화면에 데이터 기준 거래일을 표시한다.
29. 분석 범위가 KRX 전체가 아니라 종가 1,000원 이상 및 거래량 상위 200개 기준임을 표시한다.
30. 골든크로스가 매수 추천이나 상승 보장이 아님을 사용자에게 오해 없이 표시한다.

## 3. 구현하지 않는 기능

1. KRX 전체 종목을 기준으로 한 MVP 통계를 제공하지 않는다.
2. 골든크로스를 매수 추천 또는 상승 보장으로 표현하지 않는다.
3. Spring API 또는 프론트엔드에서 기술지표를 재계산하지 않는다.
4. 사용자에게 이전 리비전 비교 화면을 제공하지 않는다.
5. 휴장일을 실패 리포트로 생성하지 않는다.
6. AI 실패를 이유로 금융 데이터 리포트 공개를 중단하지 않는다.
7. Python 배치가 실행 중 DB 스키마를 생성하거나 변경하지 않는다.
8. MVP에서 인증 기반 개인화 기능을 제공하지 않는다.
9. MVP에서 운영 백업과 다중 서버 구성의 상세 정책을 구현하지 않는다.

## 4. 추후 확장 기능

1. 분석 종목 수를 200개보다 크게 확대한다.
2. KRX 전체 종목 분석을 제공한다.
3. 추가 기술지표를 도입한다.
4. Stoch MACD 외 추가 신호를 도입한다.
5. 계산 버전 v2 이상을 추가한다.
6. 종목별 AI 설명을 제공한다.
7. 로그인 기반 관심종목 저장을 제공한다.
8. 기기 간 관심종목 동기화를 제공한다.
9. 외국인 및 기관 수급 데이터를 연동한다.
10. 네이버 또는 다른 데이터 소스 fallback을 제공한다.
11. 실시간 현재가와 장중 데이터를 제공한다.
12. 실시간 호가와 체결 스트림을 제공한다.
13. 분봉 차트를 제공한다.
14. 과거 리포트 소급 생성을 제공한다.
15. 이전 리비전 비교 및 변경 원인 조회를 제공한다.
16. 종목 수 확대에 맞춰 제한 병렬 수집을 도입한다.
17. 운영 모니터링을 고도화한다.
18. 외부 백업과 보존 정책을 수립한다.
19. 다중 서버 또는 분리 배포 구성을 도입한다.
20. API 계약 버전 관리를 도입한다.
21. 상세 성능 최적화와 캐시 전략을 고도화한다.
