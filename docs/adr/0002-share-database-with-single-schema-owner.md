# 공유 데이터베이스와 단일 스키마 소유자

Spring API와 Python 배치는 하나의 PostgreSQL을 공유한다. Python은 금융 데이터 수집·분석 결과를 자신의 테이블에 적재하고, Spring은 게시 완료된 결과를 읽어 사용자 기능을 제공한다. 데이터베이스의 모든 구조 변경은 Spring의 Flyway 마이그레이션만 수행한다. 각 런타임이 별도 마이그레이션 도구를 사용하는 대안은 스키마 변경 순서와 운영 상태가 충돌할 수 있어 채택하지 않는다.

## Consequences

- Python은 Flyway가 생성한 승인된 테이블만 사용하며 실행 중 스키마를 변경하지 않는다.
- 금융 데이터·분석 결과 테이블은 Python이 쓰고 Spring이 읽는다.
- 애플리케이션 테이블은 Spring이 소유하며 Python이 접근하지 않는다.
- 스키마 변경은 Spring 저장소의 Flyway 이력에서 한 번만 관리한다.
