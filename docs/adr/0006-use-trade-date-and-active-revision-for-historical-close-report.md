# 과거 장 마감 리포트 조회 기준은 거래일과 활성 리비전으로 고정한다

과거 장 마감 리포트 조회 API의 공개 식별자는 리비전 ID가 아니라 `tradeDate`로 고정하고, Spring은 해당 거래일의 활성 리비전을 내부에서 해석한다. 응답의 공개 상태는 `PUBLISHED`, `MARKET_CLOSED`, `NOT_PUBLISHED`, `BEFORE_SERVICE_START`로 제한하며 내부 배치 진행 상태는 과거 조회 API에 세분화해 노출하지 않는다.

서비스 운영 시작일은 필수 Spring 설정 프로퍼티로 두고, 누락되면 애플리케이션이 기동하지 않게 한다. 시장 AI 요약은 거래일별 1건으로 유지하되 활성 리비전과 기준이 맞는 경우에만 완료된 요약으로 취급하며, 기존 최신 장 마감 리포트 API 계약은 변경하지 않는다.

서비스 운영 시작일의 운영 기준값은 `2026-07-01`로 둔다. 상태 판정은 서비스 시작일 이전 여부를 먼저 확인하고, 그 다음 활성 리비전 존재 여부를 확인한다. 서비스 시작일 당일은 서비스 시작 전이 아니며, 활성 리비전이 없으면 휴장 또는 미공개 상태로 판단한다.

현재 데이터 모델에서 `batch_job_run`은 거래일별 현재 배치 상태 1건만 표현한다. 과거 조회 API는 해당 거래일의 단일 `batch_job_run.status`만 휴장 여부 판단에 사용하며, 여러 실행 이력이나 최신 실행 행 선택 문제는 이번 API의 범위로 다루지 않는다.

과거 조회 API는 사용자 공개 상태를 단순하게 유지한다. `NOT_PUBLISHED`에 대한 내부 원인 필드, 서비스 시작일 이전 데이터 불일치 경고 로그, 시장 지수 누락에 따른 상태 변경, 골든크로스 데이터 부재와 0건의 구분은 이번 범위에 포함하지 않는다.
