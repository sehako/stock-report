# 종목 단위 배치 장애 격리

Python 배치는 분석 종목을 독립적으로 처리하고 종목별 처리 상태를 저장한다. 한 종목의 수집 또는 분석 실패는 다른 종목의 처리와 장 마감 리포트 공개를 막지 않는다. 모든 종목의 성공을 전체 게시 조건으로 삼는 대안은 한 외부 호출의 실패가 전체 결과를 지연시키므로 채택하지 않는다.

## Consequences

- 종목별로 신호 발견, 신호 없음, 분석 데이터 부족, 데이터 준비 중, 데이터 갱신 실패, 당일 거래 없음 및 분석 실패를 구분한다.
- 실패한 종목만 다시 처리한다.
- 시장·업종 집계에는 대상 종목 수와 실제 반영 종목 수를 함께 저장하고 표시한다.
- 누락 종목이 이후 성공하면 해당 거래일의 새 리비전으로 집계 결과를 공개한다.
