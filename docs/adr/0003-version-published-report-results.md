# 공개된 리포트 결과의 리비전 관리

원본 일봉은 종목과 거래일별로 한 행만 유지하고, 공개된 분석 결과가 자동 재시도나 데이터 정정으로 달라질 때는 기존 결과를 덮어쓰지 않고 같은 거래일의 새 리비전을 생성한다. 최초 수집 결과를 첫 리비전으로 공개하고, 실패 종목의 자동 재시도 결과는 모든 재시도가 끝난 뒤 한 번의 최종 리비전으로 모아 공개한다.

## Consequences

- Spring API는 거래일별 활성 리비전만 사용자에게 제공한다.
- 이전 리비전은 비교, 원인 추적 및 복구를 위해 보존한다.
- 미완료 리비전은 활성화하지 않으며 직전 활성 리비전을 계속 제공한다.
- 리비전 생성은 원본 일봉을 복제하지 않고 분석 결과에만 적용한다.
- 자동 재시도가 성공할 때마다 리비전을 만들지 않고 재시도 종료 후 한 번만 만든다.
- AI 시장 요약의 상태와 내용은 금융 데이터 리비전과 독립적으로 갱신하며, AI 완료나 재시도만으로 새 리비전을 만들지 않는다.
