## 개요

Python 배치에 특정 종목 또는 특정 지수의 특정 기간만 재수집할 수 있는 실행 옵션을 추가한다.

## 작업 범위

- [x] `--stock-code` 옵션을 지원한다.
- [x] `--index-code` 옵션을 지원한다.
- [x] `--start-date`, `--end-date` 기간 옵션을 지원한다.
- [x] 종목 코드와 기간 조합으로 종목 일봉을 재수집한다.
- [x] 지수 코드와 기간 조합으로 지수 일봉을 재수집한다.
- [x] 옵션을 지정하지 않으면 기본 실행 흐름으로 동작하게 유지한다.
- [x] 지원하지 않는 옵션 조합이나 잘못된 날짜 형식은 명확한 오류로 처리한다.
- [x] 기간 옵션은 `--stock-code` 또는 `--index-code`와 함께 지정한 경우에만 허용한다.
- [x] `--stock-code`와 `--index-code`를 동시에 지정하면 오류로 처리한다.
- [x] `--stock-code`는 6자리 숫자 문자열만 허용한다.
- [x] `--index-code`는 대소문자를 허용하되 내부적으로 `KOSPI`, `KOSDAQ`으로 정규화한다.
- [x] 지정 단건 실행에서 저장 가능한 row가 0건이면 오류로 처리한다.

## 검증 흐름

- [x] `--stock-code 005930 --start-date 2024-01-01 --end-date 2024-01-31` 실행 시 지정 종목과 지정 기간 데이터만 upsert되는지 확인한다.
- [x] `--index-code kospi --start-date 2024-01-01 --end-date 2024-01-31` 실행 시 `KOSPI` 지수와 지정 기간 데이터만 upsert되는지 확인한다.
- [x] 기간만 지정하거나 `--stock-code`와 `--index-code`를 동시에 지정하면 외부 FDR 호출이나 DB 쓰기 전에 오류로 처리되는지 확인한다.

## 제외 범위

- 배치 실행 이력 테이블 저장은 제외한다.
- 관리자 화면 또는 배치 실패 관리 화면은 제외한다.
- Spring Boot 조회 API 구현은 제외한다.
